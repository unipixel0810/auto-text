# A/B 테스트 설계서 — 랜딩페이지 CTA 전환율 최적화

## 1. 가설 (If-Then-Because)

### 실험 1: CTA 버튼 텍스트
> **If** CTA 버튼 텍스트를 "지금 무료로 시작하기"(A)에서 "30초만에 무료 체험하기"(B)로 변경하면,
> **Then** 클릭 전환율(CTR)이 현재 5.3%에서 10% 이상으로 상승할 것이다.
> **Because** "30초"라는 구체적 시간 제시가 진입 장벽을 낮추고, "체험"이라는 단어가 부담 없는 시도를 유도하기 때문이다.

### 실험 2: 서브 헤드라인
> **If** 서브 헤드라인을 기능 설명(A)에서 사회적 증거(B: "매달 100만 뷰를 달성하는 크리에이터들의...")로 변경하면,
> **Then** CTA 클릭 전환율이 15% 이상 리프트될 것이다.
> **Because** 사회적 증거(Social Proof)가 서비스 신뢰도를 높이고, 구체적 수치("100만 뷰")가 기대 결과를 명확히 전달하기 때문이다.

## 2. 변수 정의

| 구분 | 항목 |
|------|------|
| **독립변수** (조작) | CTA 버튼 텍스트, 서브 헤드라인 문구 |
| **종속변수** (측정) | 클릭 전환율(CTR) = 클릭 수 / 노출 수 × 100 |
| **통제변수** (고정) | 페이지 레이아웃, 색상, 폰트, 버튼 크기·위치, 트래픽 소스 비율, 시간대 |

## 3. A안(Control)과 B안(Variant) 정의

### 실험 1: `cta-button-test`
| | A (Control) | B (Variant) |
|---|---|---|
| **CTA 텍스트** | 지금 무료로 시작하기 | 30초만에 무료 체험하기 |
| **변경 요소** | 없음 (현재 그대로) | 텍스트만 변경 |

### 실험 2: `sub-headline-test`
| | A (Control) | B (Variant) |
|---|---|---|
| **헤드라인** | 업로드만 하면 끝. AI가 예능·상황·설명 자막을 자동으로 생성하고... | 매달 100만 뷰를 달성하는 크리에이터들의 편집 비법을 AI가 분석했습니다. |
| **접근법** | 기능 중심 (Feature-driven) | 사회적 증거 (Social Proof) |

## 4. 필요 표본 크기 계산

### 공식 (Cohen's h 기반)
```
h = 2 × arcsin(√p₂) - 2 × arcsin(√p₁)

n = ((Z_α/2 + Z_β) / h)²
```

### 파라미터
| 항목 | 값 |
|------|-----|
| 현재 전환율 (p₁) | 5.3% = 0.053 |
| 목표 전환율 (p₂) | 10.0% = 0.100 |
| 유의 수준 (α) | 0.05 (양측 검정) |
| 검정력 (1-β) | 0.80 |
| Z_α/2 | 1.96 |
| Z_β | 0.84 |

### 계산
```
h = 2 × arcsin(√0.100) - 2 × arcsin(√0.053)
  = 2 × 0.3217 - 2 × 0.2326
  = 0.6434 - 0.4652
  = 0.1782

n = ((1.96 + 0.84) / 0.1782)²
  = (2.80 / 0.1782)²
  = 15.71²
  ≈ 247 (그룹당)

총 필요 표본 = 247 × 2 = 494명
```

**→ 각 그룹 최소 247명, 총 494명 필요**

## 5. 예상 실험 기간

| 항목 | 값 |
|------|-----|
| 일 평균 방문자 | 약 50명 (추정) |
| A/B 분배 비율 | 50:50 |
| 그룹당 일일 유입 | ~25명 |
| 필요 표본 (그룹당) | 247명 |
| **예상 실험 기간** | **247 ÷ 25 ≈ 10일** |

> 💡 실제 방문자 수에 따라 조정:
> - 일 100명 → 5일
> - 일 30명 → 17일
> - 일 200명 → 3일

## 6. 성공/실패 판단 기준

### 주 지표: 카이제곱 검정 p-value

| p-value | 판정 | 행동 |
|---------|------|------|
| **p < 0.01** | 매우 유의미 | B안 즉시 채택 |
| **p < 0.05** | 유의미 | B안 채택 권장 |
| **0.05 ≤ p < 0.10** | 경계선 | 표본 추가 수집 후 재검정 |
| **p ≥ 0.10** | 유의미하지 않음 | A안 유지 또는 새로운 B안 설계 |

### 보조 지표
- **95% 신뢰구간**: 전환율 차이의 CI가 0을 포함하지 않아야 함
- **리프트(Lift)**: (CTR_B - CTR_A) / CTR_A × 100 → 최소 30% 이상 목표
- **MDE (최소 감지 효과)**: 4.7%p (5.3% → 10%)

### 실패 시 후속 조치
1. B안 텍스트 변경 후 재실험
2. CTA 색상/위치 등 다른 독립변수로 실험 설계
3. 사용자 인터뷰 / 히트맵 분석 결과 반영

## 기술 구현 요약

```
랜딩페이지 (landing/page.tsx)
  └─ data-ab-test="cta-button-test"
  └─ data-ab-variant-b="30초만에 무료 체험하기"
      │
      ▼
AnalyticsProvider (initABTests)
  ├─ cookie로 variant A/B 배정 (50:50)
  ├─ B 배정 시 텍스트 자동 교체
  ├─ impression 자동 기록
  └─ click 이벤트 추적
      │
      ▼
/api/ab/track → Supabase 'ab_events' 테이블
      │
      ▼
/api/ab/results → 카이제곱 검정 + p-value
      │
      ▼
/admin/experiments → 실시간 대시보드
```
