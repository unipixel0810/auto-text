## 역할 정의

### 1. Product Manager (PM)
| 항목 | 내용 |
|------|------|
| **책임** | 제품 로드맵, 우선순위 결정, 스프린트 관리 |
| **산출물** | PRD (Product Requirements Document), 릴리즈 노트 |
| **의사결정** | 기능 범위(scope), 일정, Go/No-Go 판단 |
| **Claude 활용** | 요구사항 정리, 경쟁사 분석, 우선순위 매트릭스 생성 |

### 2. 서비스 기획자 (Service Planner)
| 항목 | 내용 |
|------|------|
| **책임** | 사용자 시나리오 설계, 화면 흐름도, 기능 명세서 |
| **산출물** | 기능 명세서, 플로우차트, 와이어프레임 초안 |
| **의사결정** | 기능 동작 방식, 엣지 케이스 처리 규칙 |
| **Claude 활용** | 시나리오 자동 생성, 엣지 케이스 도출, 명세서 초안 작성 |

### 3. UX Researcher
| 항목 | 내용 |
|------|------|
| **책임** | 사용자 행동 분석, 히트맵/세션 리플레이 해석, 사용성 테스트 |
| **산출물** | UX 리서치 리포트, 퍼소나, 사용자 여정 맵 |
| **의사결정** | 사용자 Pain Point 우선순위, A/B 테스트 설계 |
| **Claude 활용** | 애널리틱스 데이터 분석, 인사이트 추출, 리포트 자동화 |

### 4. UI Designer
| 항목 | 내용 |
|------|------|
| **책임** | 시각 디자인, 컴포넌트 설계, 디자인 시스템 관리 |
| **산출물** | UI 목업, 디자인 토큰, 인터랙션 스펙 |
| **의사결정** | 색상/타이포/레이아웃, 애니메이션 스펙, 반응형 규칙 |
| **Claude 활용** | Tailwind 클래스 생성, 색상 팔레트 제안, 접근성 검증 |

### 5. Frontend Developer
| 항목 | 내용 |
|------|------|
| **책임** | React/Next.js 컴포넌트 개발, 상태 관리, 성능 최적화 |
| **산출물** | 컴포넌트 코드, 훅(hooks), 타입 정의 |
| **의사결정** | 컴포넌트 분리 전략, 렌더링 최적화, 브라우저 호환성 |
| **Claude 활용** | 코드 생성, 리팩토링, TypeScript 타입 설계 |

### 6. Backend Engineer
| 항목 | 내용 |
|------|------|
| **책임** | API 설계/구현, DB 스키마, 서버 로직, 외부 서비스 연동 |
| **산출물** | API 라우트, DB 마이그레이션, 서비스 모듈 |
| **의사결정** | API 구조, 데이터 모델, 캐싱 전략, 보안 정책 |
| **Claude 활용** | API 코드 생성, SQL 쿼리 최적화, Supabase 연동 |

### 7. QA Engineer
| 항목 | 내용 |
|------|------|
| **책임** | 테스트 시나리오 작성, 버그 리포트, 회귀 테스트 |
| **산출물** | 테스트 케이스, 버그 리포트, 품질 체크리스트 |
| **의사결정** | 릴리즈 품질 기준, 테스트 커버리지 범위 |
| **Claude 활용** | 테스트 케이스 자동 생성, 코드 리뷰, 엣지 케이스 탐색 |

## 협업 규칙

1. **핸드오프 순서**: PM → 서비스 기획자 → UX Researcher → UI Designer → Frontend/Backend → QA
2. **모든 산출물은 마크다운으로 작성**, `docs/` 디렉토리에 타임스탬프와 함께 기록
3. **각 역할의 산출물이 다음 역할의 입력(Input)**이 됨
4. **QA 피드백은 해당 역할로 역전파** — 순환 구조
5. **의사결정 충돌 시 PM이 최종 판단**
