# A/B 테스트 기능 사용 가이드

## 📋 개요

서비스 내 주요 CTA 버튼과 헤드라인의 문구를 실험하여, 어떤 버전이 더 높은 전환율을 이끌어내는지 데이터 기반으로 의사결정할 수 있습니다.

---

## 🚀 빠른 시작

### 1. HTML 요소에 A/B 테스트 속성 추가

```html
<!-- 기본 사용법 -->
<button 
  data-ab-test="cta-signup"
  data-ab-variant-b="지금 무료로 시작하기"
>
  회원가입
</button>
```

**속성 설명:**
- `data-ab-test`: 실험 이름 (고유 식별자)
- `data-ab-variant-b`: B 버전에서 표시할 텍스트

### 2. 실험 결과 확인

`/admin/experiments` 페이지에서 실시간으로 실험 결과를 확인할 수 있습니다.

---

## 📊 실험 예시

### 예시 1: CTA 버튼 테스트

```html
<!-- 헤더의 Export 버튼 -->
<button 
  data-ab-test="header-export-btn"
  data-ab-variant-b="내보내기 시작"
>
  Export
</button>
```

**결과:**
- A 버전: "Export" (기존 텍스트)
- B 버전: "내보내기 시작" (새 텍스트)
- 자동으로 노출수, 클릭수, 전환율(CTR) 추적

### 예시 2: 헤드라인 테스트

```html
<h1 
  data-ab-test="landing-headline"
  data-ab-variant-b="당신의 영상을 더욱 멋지게"
>
  영상 편집을 시작하세요
</h1>
```

### 예시 3: 복잡한 구조의 버튼

```html
<button 
  data-ab-test="premium-upgrade"
  data-ab-variant-b="프리미엄으로 업그레이드"
  className="flex items-center gap-2"
>
  <span className="material-icons">star</span>
  <span>Upgrade</span>
</button>
```

**주의:** 복잡한 구조의 경우, 마지막 텍스트 노드 또는 `<span>` 내부의 텍스트가 변경됩니다.

## 📈 대시보드 기능

### `/admin/experiments` 페이지

1. **실험 목록**
   - 활성화된 모든 실험 표시
   - 각 실험의 A/B 버전 비교

2. **통계 지표**
   - **Views (노출수)**: 각 버전이 노출된 횟수
   - **Clicks (클릭수)**: 각 버전이 클릭된 횟수
   - **CTR (%)**: 클릭 전환율 (Clicks / Views × 100)

3. **통계적 유의성 검증**
   - **카이제곱 검정** 자동 수행
   - **p-value** 표시
   - p < 0.05일 때 "통계적으로 유의미!" 배지 표시
   - 승리 variant 하이라이트

4. **AI 인사이트 분석**
   - "AI 분석하기" 버튼 클릭 시 Gemini API를 통한 분석
   - 실험 결과에 대한 구체적인 개선 방향 제시
   - 비용 최적화: 버튼 클릭 시에만 API 호출

## 🔧 기술적 세부사항

### 분배 로직

- **Cookie 기반**: `ab-variant-[experiment-name]` 형식의 쿠키 저장
- **일관성 보장**: 한 번 배정된 variant는 30일간 유지
- **50:50 분배**: 기본적으로 랜덤 배정 (향후 설정 가능)

### 이벤트 추적

- **Impression (노출)**: 페이지 로드 시 자동 기록
- **Click (클릭)**: 요소 클릭 시 자동 기록
- **Supabase 저장**: `ab_events` 테이블에 저장

### 성능 최적화

- **클라이언트 사이드 처리**: 최소한의 스크립트로 텍스트 치환
- **Lazy Loading**: 페이지 로드 시 한 번만 실행
- **중복 방지**: `data-ab-initialized` 속성으로 중복 처리 방지

## 🎯 베스트 프랙티스

1. **명확한 실험 이름 사용**
   - 예: `cta-signup`, `landing-headline`, `pricing-button`
   - 중복되지 않는 고유한 이름 사용

2. **의미 있는 차이 만들기**
   - 단순한 단어 변경보다는 사용자 행동을 유도하는 문구 사용
   - 예: "구매하기" → "지금 바로 시작하기"

3. **충분한 샘플 사이즈**
   - 최소 100회 이상의 노출 권장
   - 통계적 유의성을 확보하기 위해 충분한 데이터 수집

4. **한 번에 하나씩**
   - 여러 요소를 동시에 테스트하면 어떤 요소가 영향을 미쳤는지 알기 어려움
   - 한 번에 하나의 요소만 테스트

## 🔍 문제 해결

### 텍스트가 변경되지 않아요

1. 페이지 새로고침 확인
2. 브라우저 개발자 도구에서 쿠키 확인: `ab-variant-[실험명]`
3. `data-ab-variant-b` 속성이 올바르게 설정되었는지 확인

### 데이터가 수집되지 않아요

1. Supabase `ab_events` 테이블이 생성되었는지 확인
2. 브라우저 콘솔에서 오류 메시지 확인
3. 네트워크 탭에서 `/api/analytics/track-ab` 요청 확인

### AI 분석이 작동하지 않아요

1. `.env` 파일에 `GEMINI_API_KEY` 설정 확인
2. API 키가 유효한지 확인
3. 네트워크 탭에서 `/api/ai/analyze-ab` 요청 확인
