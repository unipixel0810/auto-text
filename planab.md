# A/B 테스트 기능 개선 계획 (planab.md)

`abtest.md` 분석 결과를 바탕으로, 기능 정상화 및 안정성을 위한 개선 계획을 수립합니다.

## 1. 데이터 조회 로직 개편 (Backend)
- **현재 문제**: 이벤트(`ab_events`)가 있는 실험만 대시보드에 노출됨.
- **개선안**: 
    - `src/lib/analytics/store.ts`의 `getABExperimentResults()` 함수 수정.
    - `ab_experiments` 테이블에서 모든 활성 실험을 먼저 가져온 뒤, `ab_events` 데이터를 매핑(Join 효과).
    - 방문자가 0명인 실험도 "진행 중(데이터 수집 중)" 상태로 리스트에 노출.

## 2. 인프라 및 환경 설정 검증
- **Supabase 연결 안정화**: `.env.local`에 설정된 키가 유효한지 체크하는 유틸리티 강화.
- **스키마 자동 확인**: 테이블(`ab_experiments`, `ab_events`) 존재 여부를 확인하고, 없을 경우 관리자에게 알림 또는 생성 가이드 제공.

## 3. 트래킹 배제 로직 조정
- **현재 문제**: `/admin` 경로가 트래킹에서 제외되어 관리자 페이지 내 실험이 불가능할 수 있음.
- **개선안**: 
    - 실험 설정 시 타겟 URL이 `/admin`을 포함하더라도 트래킹이 가능하도록 필터 옵션 조정.
    - 단, 운영 환경과 테스트 환경을 분리하기 위해 `analytics_preview` 모드 활용 권장.

## 4. 모니터링 및 상태 표시 (UI)
- **실시간 상태**: 각 실험 카드에 "데이터 수집 중", "데이터 없음", "완료" 등의 상태 뱃지 추가.
- **오류 감지**: Supabase 연결 오류 시 대시보드 상단에 경고 배너 출력.

---
## 검증 계획
- **시나리오 1**: 새 실험 생성 후 방문자 없이 대시보드 노출 확인.
- **시나리오 2**: 실제 페이지(예: `/`) 접속 후 클릭 이벤트 발생 시 수치 업데이트 확인.
- **시나리오 3**: 환경 변수 제거 후 대시보드 접속 시 안전한 오류 메시지 표출 확인.
